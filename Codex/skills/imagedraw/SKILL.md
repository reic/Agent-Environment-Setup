# 生圖 / 繪圖（Codex 版）

Codex 可依目前環境能力選擇合適的圖片產出方式。開始前先確認使用者要的是「可編輯圖表」、「網站/文件用圖片素材」，還是「照片級/插畫級點陣圖片」。

## 三條路線

| 路線 | 適合情境 | 做法 | 需求 |
|---|---|---|---|
| A：Codex 內建圖片技能 | 照片、插畫、材質、貼圖、社群圖、簡報視覺 | 使用可用的 imagegen / 圖片生成技能建立點陣圖 | 目前 Codex 環境需提供圖片生成能力 |
| B：向量圖 / 圖表 | 架構圖、流程圖、icon、簡單示意圖、Mermaid 圖 | 直接建立 SVG、Mermaid、HTML/CSS/canvas 或文件內圖表 | 不需 API key，適合可編輯輸出 |
| C：外部 API / MCP | 特定模型、公司內部工具、既有圖片管線 | 透過已設定的 MCP、Connector 或 API 腳本呼叫 | 需使用者提供並妥善管理金鑰 |

## 開始前確認

請先釐清：

- 用途：網站、簡報、文章、社群、產品 mockup、icon、流程圖。
- 尺寸：例如 1024x1024、16:9、4:3、A4、透明背景。
- 風格：寫實、扁平、科技感、手繪、商務、雜誌圖像。
- 內容：主體、場景、必要元素、不可出現元素。
- 文字：是否需要圖片內文字。重要中文文字建議後製。
- 輸出位置：例如 `assets/`、`public/images/`、文件同層資料夾。

## 建議提示格式


## Codex 操作原則

- 若使用者要點陣圖片，優先檢查目前是否有可用的圖片生成技能或工具。
- 若使用者要架構圖、流程圖或可編輯圖示，優先使用 SVG、Mermaid 或原生文件圖表。
- 不把 API key、token 或私密 prompt 寫進 repo。
- 專案圖片建議放在 `assets/`、`public/images/` 或文件指定素材資料夾。
- 交付前確認檔案存在、可開啟，並回報輸出路徑。
